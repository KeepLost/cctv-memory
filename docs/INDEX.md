# Documentation Index

## Start Here

- `CONTEXT_MANIFEST.md`: authority order, must-read documents, and conflict handling.
- `ARCHITECTURE_CONSTITUTION.md`: non-negotiable architecture rules.
- `DEVELOPMENT.md`: local setup, quality gates, migrations, and development workflow.
- `USAGE.md`: current verified user/operator usage.
- `SERVER_CLIENT_BOUNDARY.md`: server/client responsibility split.

## Normative Contracts And Specs

- `contracts/`: schemas, repository ports, authorization, search, database, configuration, API routes, testing, VLM analysis, job state machine, table schema, and non-functional requirements.

## Durable Design References

- `design/`: architecture and API design rationale that supports the contracts.
- `operations/`: runbooks and operational guidance.
- `baselines/`: benchmark and performance reference material.
- `prompts/`: prompt reference templates and prompt-version notes.
- `test-data/`: provenance for local test fixtures.
- `adr/`: ADR index. No formal ADR records exist yet.

## Historical Material

Completed investigations, old implementation reports, superseded plans, incidents, and old task specs are preserved under `status/archive/`. Treat them as evidence, not current authority.
