# Contracts Index

This directory contains durable normative contracts and specs. These files constrain future implementation unless superseded by a reviewed contract update.

- `api-routes.md`: HTTP route inventory and implementation status.
- `authorization-policy-contract.md`: principals, capabilities, access policies, and authorized scopes.
- `backup-export-contract.md`: backup/export semantics.
- `configuration-contract.md`: configuration shape, source precedence, validation, and safety rules.
- `database-adapter-contract.md`: infrastructure adapter requirements.
- `database-capability-contract.md`: database capabilities exposed to upper layers.
- `error-code-contract.md`: public error shape and vocabulary.
- `job-state-machine-contract.md`: analysis job, scale task, unit, and queue state rules.
- `module-map.md`: package/module responsibilities.
- `nonfunctional-requirements.md`: scale, latency, storage, availability, and safety targets.
- `pipeline-experiment-contract.md`: allowed experiment axes and result compatibility rules.
- `repository-port-contract.md`: repository interfaces and transaction semantics.
- `schema-contracts.md`: DTO/schema rules and canonical types.
- `search-contract.md`: search workflow and authorization-bounded retrieval rules.
- `table-schema-spec.md`: logical table schema and backend physical mappings.
- `testing-contract.md`: test layers, gates, and safety regression expectations.
- `vlm-analysis-contract.md`: video-analysis input/output, scales, prompts, publication, and failure semantics.
