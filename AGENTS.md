# CCTV Memory Agent Guide

本文件只记录项目级、长期有效的协作约束。架构、流程、API、数据格式、配置、迁移、运行方式和测试细节以 `docs/` 下的正式文档与契约为准，不在这里复述。

开始任何非微小改动前，先读 `docs/INDEX.md`。它是项目文档入口，说明哪些文档负责哪些主题；引用它是为了避免在本文件复制会漂移的实现说明，并确保后续判断回到对应契约、设计或运行文档。

## 项目定位

本项目是面向大规模视频监控分析的服务端实现。所有功能都应围绕大规模视频的分析、检索、授权访问、部署和维护展开。

项目主干只有两条：视频分析和视频检索。新增功能应明确服务于哪条主干，或说明它如何支撑二者。

## 视频分析

视频分析要平衡实时性和准确性。项目已采用运动检测、目标检测和 VLM 解析；未来可以增加更多手段，但应保持可组合，不把单一方法固化为唯一判断来源。

低成本方法适合过滤、触发、缩小候选范围或补充结构化线索；高成本方法适合语义理解。不要让所有片段无条件走最昂贵路径，也不要为了速度跳过必要的语义验证。

分析结果不是一次性产物，必须服务于后续授权检索和回放定位。新增输出时，同步考虑可检索性、可解释性、权限继承和文档化。

## 视频检索

检索必须先做权限过滤。嵌入、重排、全文检索、结构化过滤、缓存和聚合都只能在授权候选范围内工作。

权限过滤之上，应提供可组合策略，让外部 AI Agent 自由组合检索方式，而不是绑定到固定流程。

缓存机制应支持并行检索、细化、复用和比较多个方向，从而加快大规模视频检索。

检索结果面向客户端和外部 AI Agent 的组合使用，不应泄露内部存储、未授权候选、权限细节或部署秘密。

## 部署与客户端边界

实际部署需要客户端，但当前客户端尚未实现，只预留位置。大规模视频分析和检索应通过 HTTP/HTTPS 服务承载；CLI 只用于本地调试、运维和验证。

权限模型应支持未来客户端托管一部分认证或上下文能力，以减少重复认证并改善交互速度。但服务端仍必须保证未授权数据不可被检索、返回或推断。

## 依赖与测试

Python 依赖使用 `uv` 管理，不要绕过项目依赖管理。

开发时按改动类型选择测试层级：单元、契约、安全、搜索、集成、迁移、架构约束和 CLI 调试路径。具体命令、范围和现行要求见 `docs/INDEX.md` 指向的开发与测试文档。

真实外部 provider、真实数据库后端和大规模数据验证必须显式启用。默认测试路径应可重复、离线、不依赖外部服务状态。

## 文档一致性

改变架构、流程、API、数据格式、配置、迁移、权限语义、检索语义或部署假设时，必须同步更新对应文档。

长期规则不要只留在代码注释、临时记录或对话中；应进入正式文档、契约或经确认的决策记录。

当文档和代码不一致时，回到 `docs/INDEX.md` 指向的正式文档和契约，修正文档或实现，使二者重新一致。
